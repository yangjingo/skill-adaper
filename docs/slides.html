为什么做

项目动机

  • 公共 Skill 与私有工程环境之间存在“上下文断层”。
  • 用户在会话中的纠错经验，难以沉淀为可复用的 Skill 优化。
  • Skill 质量缺少可量化评估，改动效果不透明。
  • 分享前缺少统一安全扫描，存在潜在风险。
解决什么问题

核心问题与方案

上下文错配

通过 Workspace 感知分析，把 Skill 建议对齐到当前项目语言、工具链与结构。

经验流失

基于 Session 驱动演进，将实际纠错和反复指令沉淀为可追踪优化。

优化不可证

用成本、效率、摩擦、上下文负载等指标评估改动收益。

分享不安全

导入/分享环节内置安全扫描，降低恶意脚本与高危命令风险。

功能总览

这个项目能做什么

`sa import`

发现热点 Skill、导入本地/平台 Skill，支持推荐提示。

`sa info`

查看多平台 Skill 清单与详情，辅助管理与排查。

`sa evolve`

运行演进分析，并可选择应用建议变更。

`sa summary`

输出演进指标，帮助比较版本效果。

`sa scan`

扫描危险命令、提权、外传等模式，强化供应链安全。

`sa share` / `sa export`

打包、导出、PR 分享与生态协作。

命令速览

一套完整生命周期命令

引入

sa import / sa info

优化

sa evolve / sa summary

风控

sa scan / sa scan --repair

传播

sa share / sa export

定位：把“发现-改进-验证-分享”串成一条可重复链路。

应用场景

适用于哪些场景

  • 团队内部标准化：统一 Skill 质量门槛与共享流程。
  • 个人效率优化：让常用 Skill 快速贴合当前项目栈。
  • 平台迁移/并行：在 Claude Code 与 OpenClaw 间做统一治理。
  • 社区协作：将改进过的 Skill 打包、导出、提 PR 贡献上游。
  • 安全审计前置：在导入和分享前做风险筛查。
